Restaurant App

A modern, cloud-native restaurant management platform built with Flask, AWS, and Docker.
Seamlessly manage menus, orders, and notifications with a beautiful UI and robust backend.

Key Features

  • Secure user registration & login with strong password policies
  • Dynamic menu browsing with images (AWS S3)
  • Real-time order placement and order history
  • Admin panel for menu and order management
  • Instant order notifications via AWS SNS
  • Robust error handling and session security
  • Responsive, user-friendly UI with modals and AJAX

Technologies Used

Flask (Python)
AWS DynamoDB
AWS S3
AWS SNS
Docker
Gunicorn
DynamoDB
CI/CD (GitHub, AWS CodePipeline/CodeBuild)

Major Functionalities

  • User Authentication: Secure registration, login, session management, and password strength enforcement.
  • Menu Management: View, add, and delete menu items (admin only), with images stored in S3.
  • Order Placement: Users can place orders, which are stored in DynamoDB with username and timestamp.
  • Order History: Users can view their previous orders in a dedicated history section.
  • Admin Panel: Admins can view all orders, update order status, and manage menu items.
  • Notifications: Order placement triggers an SNS notification (email/SMS integration possible).
  • Error Handling: All routes have robust error handling and user-friendly feedback.

Deployment & Production

  • Production-ready Dockerfile using gunicorn as the WSGI server
  • CI/CD pipeline (AWS CodePipeline/CodeBuild) for automated build, test, and deployment
  • Deployed on AWS ECS (EC2 launch type for free tier)
  • Images stored in AWS ECR (free up to 500MB/month)
  • Environment variables and secrets managed securely

Security Best Practices

  • Strong, random Flask secret key
  • Session management and CSRF protection
  • Input validation and password hashing
  • Minimal IAM permissions for AWS resources

How to Run Locally

  1. Clone the repository
  2. Install Docker and run:
    docker build -t restaurant-app .
  3. Start the container:
    docker run -p 8000:8000 restaurant-app
  4. Visit http://localhost:8000

How to Deploy on AWS (Summary)

  1. Push code to GitHub/CodeCommit
  2. Set up AWS ECR and push Docker image
  3. Configure ECS cluster (EC2 launch type for free tier)
  4. Set up CodePipeline/CodeBuild for CI/CD
  5. Access your app via EC2 public IP or load balancer
Tip: For best performance and security, always use environment variables for secrets and configure IAM roles with least privilege.
